# AI 거짓말탐지기 PRD

## 1. 제품 요약

**서비스명:** AI 거짓말탐지기

AI 거짓말탐지기는 한 공간에 있는 A와 B가 하나의 디바이스에서 함께 진행하는 웹 기반 커플/친구용 엔터테인먼트 서비스다. A가 B에게 화면을 보이지 않은 상태로 질문을 입력하고, B가 같은 디바이스의 웹캠/전면 카메라 앞에서 답하면 AI가 영상, 음성, 답변 텍스트, 로컬 행동 feature를 종합해 결과를 **진실** 또는 **거짓**으로 단정 출력한다.

핵심 경험은 "정확한 법과학 판정"이 아니라, 사용자가 카메라 앞에서 검사받는 듯한 몰입감, 결과 공개의 긴장감, 그리고 SNS에 올리고 싶은 릴스형 영상 내보내기다.

## 2. 목표

- 웹 MVP를 빠르게 출시한다.
- 첫 버전은 결제 없이 무료로 제공한다.
- Supabase Free, Vercel Hobby, Cloudflare R2 Free를 전제로 운영 비용을 낮춘다.
- 추후 Polar 결제와 앱인토스 IAP/광고를 붙일 수 있도록 권한/결제 구조를 분리한다.
- 분석 결과는 공유하기 쉬운 짧고 강한 포맷으로 만든다.
- 카카오 로그인을 전제로 사용자 세션과 무료 사용 횟수를 관리한다.
- 모바일 뷰와 PC 뷰를 모두 지원한다.

## 3. 비목표

- 결과 화면에서 확률, 가능성, 감지 신호, 내부 점수를 공개하지 않는다.
- "왜 거짓인지"를 설명하지 않는다.
- 질문 자동 생성 유료화는 MVP에 넣지 않는다.
- 서버 FFmpeg 기반 영상 렌더링은 MVP에서 제외한다.
- 원본 영상 장기 보관은 하지 않는다.

## 4. 타깃 사용자

1. 커플
   - "어제 누구랑 있었어?", "나한테 숨기는 거 있어?" 같은 질문을 장난스럽게 던지고 싶은 사용자.

2. 친구/술자리/파티
   - 웃긴 질문을 던지고 결과 영상을 바로 공유하고 싶은 사용자.

3. SNS 바이럴 사용자
   - 결과보다 검사받는 화면과 리액션을 릴스/쇼츠/TikTok에 올리고 싶은 사용자.

## 5. 핵심 포지셔닝

외부 카피:

- "착한 내 남자친구, 과연 나한테 거짓말하는 게 없을까?"
- "CQT, 캘리브레이션을 통한 AI Vision 기반 전문적 진위 판단"
- "AI는 과연 거짓말을 알아챌 수 있을까?"

결과 화면의 강도:

- 헤드라인은 오직 `진실` 또는 `거짓`.
- 아래 코멘트는 유머러스하고 조롱 섞인 MZ 톤.
- 심한 욕설은 쓰지 않는다.
- 감지 신호는 절대 공개하지 않는다.

## 6. 사용자 플로우

MVP는 비동기 링크 공유가 아니라, 같은 공간에서 하나의 디바이스로 함께 진행하는 흐름을 기본 전제로 한다.

### 6.1 카카오 로그인

1. 랜딩 진입
2. 카카오 로그인
3. 무료 사용 가능 여부 확인
4. 질문 입력 화면 진입

### 6.2 A의 질문 입력

1. A가 질문을 입력한다.
2. 화면 상단에 B가 보지 않도록 안내한다.
3. A가 `질문 잠그기`를 누른다.
4. 질문 입력 내용은 숨김 처리되고, B에게 넘겨도 되는 화면으로 전환된다.

필수 UX 문구:

- "지금은 질문 입력 중이야. 상대가 화면 못 보게 해줘."
- "질문 다 썼으면 잠가줘. 잠그면 상대에게 보여줘도 돼."
- "잠금 완료. 이제 폰을 상대에게 넘겨줘."

### 6.3 B의 검사 진행

1. 카메라/마이크 권한 허용
2. 얼굴 위치, 조명, 음성 상태 체크
3. 시작 질문에 답변
4. A가 입력한 진짜 질문에 답변
5. 분석 중 화면 시청
6. 결과 확인
7. 릴스용 영상 내보내기

### 6.4 화면 대응

- 모바일에서는 한 손으로 진행 가능한 세로형 플로우를 기본으로 한다.
- PC에서는 웹캠 중앙 정렬, 좌우 분석 패널, 하단 진행 버튼을 사용한다.
- 모든 핵심 화면은 모바일 360px 폭부터 데스크톱 1440px 이상까지 깨지지 않아야 한다.

## 7. 질문 구조

MVP는 2단계 질문 구조를 사용한다.

### 7.1 시작 질문

역할:

- 평소 말투, 표정, 음성 리듬, 얼굴 추적 상태를 자연스럽게 잡는다.
- 서비스 외부/랜딩에서는 전문성을 위해 `CQT`, `캘리브레이션` 표현을 사용할 수 있다.
- 실제 진행 UI에서는 사용자가 멈칫하지 않도록 쉬운 반말 문구를 쓴다.

UI 문구:

- "먼저 가볍게 하나만 물어볼게."
- "오늘 하루 중 제일 기억나는 일 뭐야?"

내부 타입:

```ts
type QuestionType = "warmup" | "target";
```

### 7.2 진짜 질문

역할:

- A가 입력한 핵심 질문.
- 결과 판정의 중심.
- Gemini는 전체 영상, 진짜 질문 구간 5 FPS 샘플, 로컬 feature JSON, transcript를 함께 보고 판정한다.

UI 문구:

- "이제 진짜 질문이야."
- A가 입력한 질문을 그대로 노출한다.
- A가 입력한 질문은 보통 반말일 가능성이 높으므로, 주변 UI 문구도 반말 톤으로 맞춘다.

## 8. 분석 중 화면

분석 중 화면은 실제 분석 신호를 공개하지 않지만, 굉장히 전문적인 장비를 보는 듯한 느낌을 줘야 한다.

표시 가능한 모듈명:

- 얼굴 프레임 추적
- 시선 흐름 스캔
- 음성 파형 분석
- 답변 리듬 처리
- 표정 변화 맵
- 심박 신호 추정
- 응답 패턴 비교
- AI 판정 엔진
- 멀티모달 동기화

금지 문구:

- 시선 이탈 감지
- 음성 떨림 감지
- 거짓 신호 발견
- 수상한 순간
- 심박 급상승
- 확률 계산

## 9. 결과 화면

### 9.1 헤드라인

헤드라인은 아래 둘 중 하나만 허용한다.

- `진실`
- `거짓`

헤드라인에는 다른 단어, 조사, 문장부호, 이모지, 확률을 붙이지 않는다.

### 9.2 코멘트 톤

헤드라인 아래에는 roast comment를 보여준다.

예시:

거짓:

- "구라도 실력입니다 선생님. 조금 더 노력하세요."
- "노력은 가상했는데 AI가 님보다 생각보다 똑똑합니다."
- "양심은 집에 두고 왔나요? ㅋㅋ 하마터면 속을 뻔 했습니다.."

진실:

- "보기와는 다르게 생각보다 정직하신 편이네요."
- "AI가 오늘 사람 한 명 구했습니다."

### 9.3 결과에서 공개하지 않는 것

- 확률
- 가능성
- 내부 점수
- 모델 confidence
- 감지 신호
- 어떤 행동이 수상했는지
- feature별 분석 근거

### 9.4 결과에서 공개하는 것

- 질문
- 답변 요약
- `진실` 또는 `거짓`
- roast comment
- 내보내기 버튼
- 다시 하기 버튼

## 10. 공유/내보내기

### 10.1 공유 카드

공유 카드에는 질문을 공개한다.

포맷:

```text
AI 거짓말탐지기
질문: 어제 밤 10시 이후에 이성이랑 단둘이 있었어?
판정: 거짓
```

감지 신호는 포함하지 않는다.

### 10.2 릴스용 영상

기본 export:

- 9:16 세로 영상
- 질문 공개
- 답변 장면 포함
- 분석 UI 오버레이 포함
- 최종 판정은 `진실` 또는 `거짓`
- 감지 신호 미포함
- 확률 미포함

MVP에서는 브라우저에서 `canvas.captureStream()`과 `MediaRecorder`로 생성한다.

## 11. 분석 입력 전략

최종 확정안:

```text
전체 세션 영상: Gemini 1 FPS
진짜 질문 구간: Gemini 5 FPS
로컬 feature JSON: 브라우저 15~30 FPS
음성 feature: 20~50ms window
Transcript: 시작 질문 + 진짜 질문 답변
```

이유:

- Gemini 기본 1 FPS는 전체 맥락 파악에 적합하다.
- 진짜 질문 구간은 더 촘촘한 표정/움직임 해석이 필요하므로 5 FPS로 보강한다.
- rPPG, blink, head pose, 음성 리듬은 Gemini 영상 프레임보다 브라우저 feature 추출이 더 적합하다.

## 12. 수익화 로드맵

### 12.1 MVP

- 결제 없음
- 무료 1회 제한은 구현 가능하되, 강제 결제는 붙이지 않는다.
- 사용량/전환 데이터를 먼저 본다.

### 12.2 웹 결제

추후 Polar를 붙인다.

권장 상품:

- 3회권
- 5회권
- 10회 파티팩

1회권은 Polar의 고정 수수료 때문에 비효율적이다.

### 12.3 앱인토스

처음부터 앱인토스 입점을 고려해 구조를 설계한다.

앱인토스 버전 수익화:

- 무료 1회
- 보상형 광고로 1회 추가
- IAP 3회권
- IAP 5회권
- 파티팩 10회권

앱인토스에서는 디지털 이용권 성격이므로 일반 토스페이보다 인앱결제(IAP)가 자연스럽다.

## 13. 성공 지표

- 검사 시작 대비 완료율
- 결과 페이지 도달률
- 내보내기 클릭률
- 실제 영상 다운로드율
- 결과 공유율
- 한 유저당 평균 질문 수
- 무료 1회 이후 추가 사용 의향
- 앱인토스 심사용 이탈/차단 포인트 수

## 14. 운영 제약

MVP 인프라:

- Vercel Hobby
- Supabase Free
- Cloudflare R2 Free
- Cloudflare Worker Free
- Gemini API

제약:

- Vercel API를 통해 큰 영상을 업로드하지 않는다.
- Supabase Storage에 원본 영상을 저장하지 않는다.
- R2 영상은 짧게 보관한다.
- Gemini Files API 파일은 임시 분석용으로만 사용한다.
